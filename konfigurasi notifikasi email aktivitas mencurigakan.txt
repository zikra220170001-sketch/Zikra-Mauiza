Langkah 1: Aktifkan ekstensi php_zip pada composser
  Buka file C:\xampp\php\php.ini
  Cari ;extension=zip
  Hapus tanda ; di depannya, jadi: extension=zip

Langkah 2: Pastikan PHP OpenSSL aktif
  Buka file C:\xampp\php\php.ini
  Cari ;extension=openssl
  Hapus tanda ; di depannya, jadi: extension=openssl

Langkah 3: Install PHPMailer
  Buka CMD dan jalankan:
    cd C:\xampp\htdocs\mfa_crud
    composer require phpmailer/phpmailer
  Setelah instalasi selesai, pastikan file berikut muncul di proyekmu:
    Folder vendor/
    File composer.json
    File composer.lock
    File vendor/autoload.php
  Jika belum punya Composer Download di:  https://getcomposer.org/download

Langkah 4: Aktifkan App Password Gmail (2FA)
  Aktifkan “2-Step Verification” (2FA)
    Buka https://myaccount.google.com/security. Login dengan akun Gmail kamu.
    Klik 2-Step Verification (Verifikasi 2 Langkah)
    Aktifkan: Turn on 2-step Verification
  Buka Halaman App Password: https://myaccount.google.com/apppasswords
  Akan muncul halaman seperti ini:
    Select App name:  Windows Computer
  Klik tombol Generate.
  Maka Google akan menampilkan kode 16 karakter
  Masukkan ke mail_config.php dengan mengganti baris:
   $mail->Username   = 'email@gmail.com';
     $mail->Password   = 'abcdefghijklmnop'); // ← isi dengan App Password
     $mail->SMTPSecure = 'tls';
     $mail->Port       = 587;
     $mail->setFrom('email@gmail.com', 'Sistem Keamanan Login');

Langkah 5: Tambahkan Logika Blokir Otomatis Account Lock)
  Kode berfungsi menonaktifkan sementara akun pengguna yang gagal login 3 kali.
    if ($blocked_until && $current_time < $blocked_until) {

Hasil Uji Konfigurasi Keamanan Login
  Sekarang buka di browser: http://localhost/mfa_crud
  Klik Daftar Akun Baru (register.php)
  Login dengan username & password benar
  Jika berhasil masuk ke dashboard.php
  Sekarang uji gagal login berulang
  Login dengan password salah (misalnya 111) sebanyak 3 kali berturut-turut.
  Setelah percobaan ke-3, sistem akun akan dibekukan selama 2menit
  Gmail notifikasi peringatan mencurigakan akan masuk ke admin
  Setelah 2menit sistem kembali bisa diakses
  Terakhir pastikan login_logs mencatat seluruh status login di database.
